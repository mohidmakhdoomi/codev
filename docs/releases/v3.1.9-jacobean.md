# v3.1.9 Jacobean

Released: 2026-06-08

Hotfix republish of v3.1.8 with `workspace:*` dependencies resolved to concrete versions.

## What this fixes

`@cluesmith/codev@3.1.8` was published with unresolved `workspace:*` references in its `dependencies` (e.g. `"@cluesmith/codev-core": "workspace:*"`). End users running `npm install -g @cluesmith/codev@3.1.8` hit `npm error code EUNSUPPORTEDPROTOCOL` because npm doesn't understand the `workspace:` protocol — that rewriting is normally done by `pnpm publish` at publish time, and the v3.1.8 publish accidentally used `npm publish` instead.

v3.1.9 republishes the same code through `pnpm publish`, which rewrites `workspace:*` to `3.1.9` correctly. Verified post-publish: `npm view @cluesmith/codev@3.1.9 dependencies` shows `'@cluesmith/codev-core': '3.1.9'`.

## What changed code-wise

Nothing. The package contents are identical to v3.1.8 — the only difference is the dependency strings in the published `package.json`. Anyone who successfully ran `pnpm install -g @cluesmith/codev@3.1.8` (pnpm tolerates the unresolved `workspace:` protocol) is on the same code as v3.1.9 minus the bumped version field.

For the full set of v3.1.8 changes (terminals & Tower restarts story, agy CLI swap, Codev Dev surface, polish across Builders / dashboard / consult), see [v3.1.8 Jacobean](./v3.1.8-jacobean.md).

## Action required

If you ran `npm install -g @cluesmith/codev@3.1.8` and got the `EUNSUPPORTEDPROTOCOL` error:

```bash
npm install -g @cluesmith/codev@3.1.9
afx tower stop && afx tower start
```

If you successfully installed v3.1.8 with `pnpm install -g`, the upgrade to v3.1.9 is a no-op but recommended for consistency.

VS Code extension users are unaffected — the marketplace v3.1.8 ships with esbuild-bundled dependencies, so the `workspace:*` strings in its `package.json` are inert at runtime.

## Lessons learned

The v3.1.8 publish used `cd packages/codev && npm publish` which does not rewrite `workspace:*` deps. Future releases must use `pnpm publish` (or `pnpm -r publish`) for any package with workspace-protocol dependencies. The release protocol's npm-publish section has been updated to make this explicit.
